The whitepaper

Abstract

We present Synapse Protocol, a decentralized marketplace for AI inference compute settled on the Solana blockchain. The protocol enables permissionless participation on both sides of the market: any GPU owner can register as a node operator, and any developer can route inference requests to the network through an OpenAI-compatible API.

Synapse solves three structural problems with existing centralized AI compute providers: monopoly pricing, censorship of model outputs, and the economic impossibility of per-token micropayments on high-latency, high-fee blockchains.

The $SYN token serves a dual purpose: it is the payment medium for inference requests and the staking collateral for node operators. A fixed 15% of every fee is burned, creating mechanical supply deflation proportional to network demand.

The total supply of $SYN is fixed at 1,000,000,000 tokens with no additional minting possible. Governance is conducted on-chain via Solana Realms with a 1 token = 1 vote mechanism.

Protocol Architecture

The Synapse protocol consists of four on-chain programs deployed on Solana mainnet: the Router, the Registry, the Escrow, and the Verifier.

When a user submits an inference request, the Router selects the optimal node from the Registry based on a weighted score incorporating reputation, current load, and geographic latency. Payment is held in the Escrow program pending proof submission.

// Inference request structure pub struct InferenceRequest { pub model_id: Pubkey, pub input_hash: [u8; 32], pub max_tokens: u32, pub requester: Pubkey, pub fee_lamports: u64, pub timestamp: i64, }

Node operators submit an optimistic proof within 30 seconds of completing inference. The Verifier program validates the proof structure and releases 85% of the escrowed fee to the node operator. The remaining 15% is sent to the burn address.

In the event of a disputed result, any party may trigger the fraud proof window (48 hours). A successful fraud proof results in full slashing of the node operator's stake and a bounty paid to the challenger.

Token Economics

Supply mechanics: 1B SYN at genesis. Zero inflation. Every inference call destroys 15% of the fee in $SYN, permanently reducing circulating supply. At 4M daily inferences with an average fee of 0.01 SYN, approximately 6,000 SYN is burned per day — roughly 2.19M per year at current network throughput.

As AI inference demand grows with the broader market, the burn rate scales proportionally. The protocol is structurally deflationary as long as it has any usage at all.

Staking: Node operators must stake a minimum of 1,000 SYN as collateral. Staked tokens earn a share of the 85% operator fee in proportion to reputation-weighted throughput. Delegators may stake to existing nodes and receive a portion of their earnings without operating hardware.

Governance: The treasury (10% of initial supply, 100M SYN) is controlled by the DAO. Proposals require 1% quorum and 66% approval to pass. Token-weighted voting is conducted fully on-chain.

Security Model

Synapse uses an optimistic fraud proof scheme inspired by Optimism's dispute game. Results are assumed correct by default and can be challenged within a 48-hour window. This design prioritizes throughput over latency of finality, which is appropriate for AI inference where exact output determinism is verifiable but doesn't need to block the main flow.

Slashing conditions include: submitting a proof for an incorrect result, extended downtime beyond the 95% SLA threshold, and attempted double-claiming. Slashed collateral is split between the challenger bounty (50%) and the protocol treasury (50%).

The protocol has been audited by OtterSec and Neodyme. All audit reports are publicly available in the Synapse GitHub repository.

Sybil resistance: Node registration requires a minimum stake of 1,000 SYN (~$840 at launch price). This creates a meaningful economic barrier to Sybil attacks while remaining accessible to individual GPU operators.

Roadmap

Phase 1 — Testnet (Q1 2025): Deploy Router, Registry, and Escrow programs on Solana devnet. Onboard 10 whitelisted GPU node operators. Launch inference API supporting Llama 3 70B and Mistral 7B.

Phase 2 — Mainnet Alpha (Q2 2025): Permissionless node registration. ZK fraud proof verifier deployment. OpenAI-compatible SDK release. Model registry for third-party model publishing.

Phase 3 — DAO Launch (Q3 2025): Full governance activation via Solana Realms. Treasury DAO control transfer. Cross-chain bridge via Wormhole for ETH and BASE users.

Phase 4 — Fine-tuning Marketplace (Q4 2025): On-chain fine-tuning job coordination. Model creator royalties for derivative models. Reputation oracle V2 with ZK attestations.
