Deploy Synapse to GitHub Pages

Your site will be live at https://YOUR-USERNAME.github.io/synapse-app in about 2 minutes.

Create the GitHub repository

Go to github.com/new and create a new public repository named synapse-app.

Don't initialize with a README — you'll push the files directly.

Extract and push the files

Download the ZIP from the link above, then run these commands in your terminal:

mkdir synapse-app && cd synapse-appclick to copy
git initclick to copy
cp ~/Downloads/synapse-v4.html index.htmlclick to copy
touch .nojekyllclick to copy
git add .click to copy
git commit -m "Initial deploy: Synapse Protocol landing page"click to copy
git remote add origin https://github.com/YOUR-USERNAME/synapse-app.gitclick to copy
git push -u origin mainclick to copy
Replace YOUR-USERNAME with your actual GitHub username.

Enable GitHub Pages

Go to your repo → Settings → Pages

Under Source, select Deploy from a branch → choose main → / (root) → click Save.

GitHub will show a banner: "Your site is live at…" — usually takes 30–90 seconds.

Your live URL

https://YOUR-USERNAME.github.io/synapse-app

Share this link — it serves the full interactive site with wallet connect, swap modals, dashboard, and whitepaper reader.

Custom domain? Add a file named CNAME containing just app.synapse.so, push it, then point your DNS CNAME to YOUR-USERNAME.github.io.

Update the site links

Open index.html and replace the two placeholder GitHub URLs with your actual repo:

sed -i 's|synapse-protocol/app|YOUR-USERNAME/synapse-app|g' index.htmlclick to copy
git add index.html && git commit -m "Update repo links" && git pushclick to copy
